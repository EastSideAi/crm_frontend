# Омниканальный AI-бот + CRM — ТЗ (со стороны заказчика, очищено)

> Источник: бриф заказчика (EastSide). Здесь — только что нужно получить, без GPT-воды.
> CRM-часть («видеть переписку бота с клиентом» + аналитика по клиенту, в т.ч. расход AI)
> внедряется ДИЗАЙНОМ сейчас (на демо-данных), бэкенд-логика и сам бот — позже.
> Важно: из CRM НЕ переписываемся с клиентом (это не цель) — CRM показывает, КАК человек
> общается с ботом, и аналитику. Бот работает И на платформе, И отдельно (омниканально).

## Проблема
Клиенты пишут куда попало (Telegram, VK, WhatsApp, Instagram, комменты, личные аккаунты
менеджеров) → заявки теряются, отвечают медленно, хаос. Нужна единая система: ловит все
обращения, быстро отвечает, не даёт лидам слиться. Компания образовательная (поступление
Китай/Корея/Япония, языковые курсы, гранты).

## Бот умеет 5 вещей
1. Сразу ответить клиенту (не оставлять висеть).
2. Расспросить и квалифицировать — кто пишет (родитель/ученик), возраст, направление/страна,
   уровень языка, сроки, бюджет, главный вопрос.
3. Довести до целевого действия — диагностика / консультация / пробный урок / тест / оплата.
4. Отвечать на типовые вопросы по базе знаний (поступление, гранты, визы, оплата, программы).
5. Передать менеджеру, если сложно / готов платить / просит человека — с резюме диалога.

## Каналы
Бот НЕ сидит на личных аккаунтах менеджеров (банят при автоматизации — подтверждено).
- MVP: Telegram (офиц. бот), VK, WhatsApp, Сайт/форма диагностики, CRM (всё стекается сюда).
- Потом: Instagram Direct, комментарии (Insta/VK), Max, и уже после техпроверки — личные аккаунты.

## Логика
Клиент пишет → бот фиксирует канал → здоровается → квалифицирует → создаёт/обновляет карточку
лида в CRM → предлагает целевое действие → записался: подтверждение + напоминание; слился:
follow-up. Вся переписка из всех каналов → в ОДНУ карточку лида в CRM (центральное хранилище).

## Триггерные сообщения (MVP-минимум)
- написал впервые → приветствие + квалификация
- начал диагностику, не закончил → напомнить
- прошёл диагностику, не записался → предложить консультацию
- записался → подтверждение
- скоро консультация → напоминание
- не пришёл → предложить перенести
- получил предложение, не оплатил → мягкое напоминание
- сложный вопрос → передать менеджеру

## Два разных бота (не путать)
- Омниканальный AI-бот — общается с клиентами.
- Внутренний бот-уведомитель — пишет команде (новый/горячий лид, просит менеджера, готов
  платить, бот не смог ответить, ошибка интеграции). ОТДЕЛЬНЫЙ бот.

## База знаний
Бот отвечает ТОЛЬКО по утверждённой базе, при неуверенности → менеджеру. База обновляется без
переделки бота. Разделы: поступление в Китай, гранты, языковой год, бак/маг/PhD, языковые курсы,
экзамены (DET/IELTS/TOEFL/HSK), Future Discovery, Харбин/Шанхай 2026, цены, договоры,
оплата/рассрочка, документы, визы, проживание, билеты, гарантии, контакты.

## Админка (CRM)
Все диалоги, фильтры по каналам/статусам, поиск клиента, ручная смена статуса, назначение
менеджера, **отключить AI в конкретном диалоге и вернуть**, карточка клиента, шаблоны сообщений,
управление базой знаний, ошибки интеграций, аналитика.

## Аналитика
Время первого ответа, лиды по каналам, конверсии по этапам воронки, где отваливаются, сколько
передано менеджеру, сколько AI закрыл сам, частые вопросы и возражения, стоимость обработки лида.

## Технические требования (жёсткие)
- Первый ответ клиенту — до 10 сек. Карточка в CRM — до 5 сек. Передача менеджеру — моментально.
- Персональные данные и backend — на российских серверах (обработка ПД на территории РФ).
- Логирование всего, фиксация ошибок, резервное сообщение при падении AI.
- Роли: админ, РОП, менеджер, куратор, маркетолог, разработчик.

## НЕ делаем в v1
Личные Telegram-аккаунты, свой «Телетайп», глубокую автоматизацию Max, автоответы на все
комменты, voice-логику, автономного AI-продавца без контроля, авто-юрконсультации и
авто-обещания гарантий поступления.

## Критерий «готово» (MVP)
Клиент пишет в Telegram/VK/WhatsApp/сайт → бот отвечает, квалифицирует, записывает на
диагностику/консультацию → карточка в CRM с историей → менеджер получает уведомление о горячем
лиде → бот умеет отдать диалог человеку → отвечает по базе знаний → ≥5 триггеров → базовая
аналитика, логи ошибок, кнопка отключения AI.

## Что делаем в CRM ПРЯМО СЕЙЧАС (только дизайн, демо-данные)
1. Страница «Диалоги» — омниканальный инбокс: список переписок по каналам, фильтры канал/статус,
   поиск, индикатор «AI вкл/выкл», превью последнего сообщения, статус (ждёт ответа / AI ведёт /
   передан менеджеру). Клик → карточка клиента, раздел «Диалог».
2. Раздел «Диалог» в карточке клиента — омниканальная переписка (бот/клиент/менеджер, тег канала),
   переключатель «AI ведёт диалог» (вкл/выкл), строка расхода AI (₽/сообщения/токены).
3. Страница «Аналитика бота» — первый ответ, лиды по каналам, % закрытых AI, передано менеджеру,
   расход AI, конверсии, частые вопросы. (Демо-данные, помечено баннером.)
Бэкенд и подключение бота — следующей итерацией; UI готовится под контракт сейчас.
